# Проект анализа дефектов в программном обеспечении (Software Defect Prediction)

## Описание проекта

Данный проект представляет собой полноценный **ETL-конвейер** для обработки и анализа данных о дефектах в программном обеспечении. Проект включает в себя загрузку с Google Drive, очистку, трансформацию и сохранение данных в CSV формате, а также исследовательский анализ данных (EDA) для выявления факторов, влияющих на появление ошибок в коде.

### Ключевые возможности

*   **CLI интерфейс** для гибкого управления ETL процессом
*   **Загрузка с Google Drive** по ID файла
*   **Конфигурация через `.env`** файл
*   **Пошаговое выполнение** этапов ETL (извлечение, трансформация, загрузка)
*   **Валидация и очистка** метрик программного кода
*   **Jupyter Notebook** для визуального анализа и построения графиков

### Структура данных

Набор данных содержит статические метрики кода, собранные из программных модулей, включая:

*   **Базовые метрики:** количество строк кода (`loc`)
*   **Метрики сложности:** цикломатическая сложность (`cyclo_complexity`)
*   **Метрики Холстеда:** объем (`volume`), сложность (`difficulty`), усилия (`effort`)
*   **Структурные метрики:** количество ветвлений (`branch_count`), условий (`condition_count`)
*   **Целевая переменная:** наличие или отсутствие дефекта (`defect_label`)

## ETL-пакет для обработки данных о кофе

Проект реализует полноценный **ETL-конвейер**, состоящий из трёх модулей:

- **`extract.py`** — загрузка исходных данных из локального CSV-файла или Google Drive
- **`transform.py`** — очистка, нормализация и валидация данных, сохранение в `.csv` и `.parquet`
- **`load.py`** — сохранение финальных обработанных данных в CSV формате

### Диаграмма ETL процесса

```bash
python etl/main.py [путь_к_файлу]
```

### CLI интерфейс

Проект оснащён мощным CLI интерфейсом для гибкого управления ETL процессом:

#### Полный запуск конвейера

```bash
# Использовать настройки из .env (автоматически загружает с Google Drive)
python etl/main.py

# Локальный файл
python etl/main.py --local data/Coffee-modified.csv

# Загрузка с Google Drive напрямую
python etl/main.py --gdrive "https://drive.google.com/file/d/1O3GpKitaVD4jjsqNiwahw8qU2ARUxDMJ/view"
```

#### Пошаговое выполнение

```bash
# Только извлечение данных
python etl/main.py --extract-only

# Только трансформация (требует предварительного extract)
python etl/main.py --transform-only

# Только загрузка в финальное хранилище
python etl/main.py --load-only
```

#### Дополнительные опции

```bash
# Подробный вывод с трейсбеком ошибок
python etl/main.py --verbose

# Указать путь для сохранения загруженного файла
python etl/main.py --gdrive "URL" --output data/my_data.csv

# Справка по всем командам
python etl/main.py --help

# Версия программы
python etl/main.py --version
```

## Функциональность ETL

### Extract (Извлечение)
- Загрузка CSV-файла с данными о кофе
- Первичная проверка корректности данных
- Сохранение данных в `data/raw/raw_coffee_data.csv`

### Transform (Трансформация)
- Очистка и нормализация названий столбцов
- Обработка текстовых столбцов (приведение к нижнему регистру, удаление пробелов)
- Валидация и очистка числовых столбцов:
  - Оценки качества (0-10): Aroma, Flavor, Aftertaste, Acidity, Body, Balance, Uniformity, Clean Cup, Sweetness, Cupper Points
  - Total Cup Points (0-100)
  - Moisture (0-1)
  - Высота выращивания (0-5000 метров)
  - Количество дефектов
- Обработка пропущенных значений
- Удаление дубликатов
- Сохранение результатов в `data/processed/clean_coffee_data.csv` и `clean_coffee_data.parquet`
- Валидация выходных параметров

### Load (Сохранение)
- Чтение данных из `clean_coffee_data.parquet` (если DataFrame не передан)
- Сохранение финальных обработанных данных в `data/final/final_coffee_data.csv`
- Вывод статистики по сохраненным данным

- **Пропущенные значения:**

- Проверка целостности данных
- Проверка размера финального датасета
- Логирование статистики

## FAQ (Часто задаваемые вопросы)

### Q: Как получить файл с Google Drive?

**A:** Проект автоматически загружает файлы с Google Drive. Просто укажите URL в `.env` файле или передайте через CLI:

```bash
python etl/main.py --gdrive "https://drive.google.com/file/d/FILE_ID/view"
```

### Q: Что делать, если файл на Google Drive недоступен?

**A:** Убедитесь, что:
1. Файл имеет публичный доступ (или доступ по ссылке)
2. URL корректный и содержит `file_id`
3. Есть интернет-соединение

Как альтернатива, используйте локальный файл:

```bash
python etl/main.py --local data/Coffee-modified.csv
```

### Q: Можно ли запустить только часть ETL процесса?

**A:** Да! Используйте флаги:
- `--extract-only` — только загрузка данных
- `--transform-only` — только трансформация
- `--load-only` — только сохранение

### Q: Где хранятся промежуточные результаты?

**A:** 
- `data/raw/` — сырые данные после Extract
- `data/processed/` — очищенные данные после Transform (CSV и Parquet)
- `data/final/` — финальные данные после Load

### Q: Как подключить базу данных?

**A:** Настройте параметры БД в файле `.env`:

```bash
DB_HOST=localhost
DB_PORT=5432
DB_NAME=coffee_db
DB_USER=your_username
DB_PASSWORD=your_password
```

### Q: Проект поддерживает другие форматы данных?

**A:** В текущей версии поддерживаются CSV и Parquet. Для других форматов требуется модификация модулей `extract.py` и `load.py`.

## Примечания

- Данные о кофе содержат информацию из различных стран и регионов мира
- Проект работает с CSV и Parquet файлами
- Все данные сохраняются локально в директории `data/`
- Рекомендуется использовать Python 3.10 или выше
- Файл `.env` автоматически добавлен в `.gitignore`
- Для работы с Google Drive требуется интернет-соединение
