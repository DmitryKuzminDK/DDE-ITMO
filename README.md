# Tech Defect Prediction - ETL Pipeline for Software Defect Analysis

## 📋 Описание проекта

ETL-конвейер и исследовательский анализ данных (EDA) для датасета предсказания дефектов в программном обеспечении. Проект позволяет загружать данные о метриках кода с Kaggle, выполнять их очистку и трансформацию, а также проводить глубокий анализ для выявления факторов, влияющих на появление дефектов.

### 🎯 Ключевые возможности

- Загрузка данных с Kaggle API или из локального файла
- Полноценный ETL-конвейер (Extract, Transform, Load)
- CLI интерфейс для гибкого управления
- Валидация и очистка метрик кода
- Детальные отчеты о трансформации
- Визуализация и анализ в Jupyter Notebook
- Подготовка данных для машинного обучения
- Автоматическая архивация данных

## 📊 Датасет

**Источник:** [Software Defect Prediction Dataset](https://www.kaggle.com/datasets/ziya07/software-defect-prediction-dataset) на Kaggle

**Описание:** Статические метрики кода из проектов NASA, размеченные на наличие/отсутствие дефектов. Содержит различные метрики, включая количество строк кода, цикломатическую сложность, метрики Холстеда и другие показатели качества кода.

### Основные признаки:
- `loc` — количество строк кода
- `cyclo_complexity` — цикломатическая сложность
- `halstead_volume` — объем по Холстеду
- `halstead_difficulty` — сложность по Холстеду
- `halstead_effort` — усилия по Холстеду
- `branch_count` — количество ветвлений
- `condition_count` — количество условий
- `defect_label` — целевая переменная (0 - нет дефекта, 1 - есть дефект)
