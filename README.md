# Проект анализа дефектов в программном обеспечении (Software Defect Prediction)

## Описание проекта

Данный проект представляет собой полноценный **ETL-конвейер** для обработки и анализа данных о дефектах в программном обеспечении. Проект включает в себя загрузку с Google Drive, очистку, трансформацию и сохранение данных в CSV формате, а также исследовательский анализ данных (EDA) для выявления факторов, влияющих на появление ошибок в коде.

### Ключевые возможности

*   **CLI интерфейс** для гибкого управления ETL процессом
*   **Загрузка с Google Drive** по ID файла
*   **Конфигурация через `.env`** файл
*   **Пошаговое выполнение** этапов ETL (извлечение, трансформация, загрузка)
*   **Валидация и очистка** метрик программного кода
*   **Jupyter Notebook** для визуального анализа и построения графиков

### Структура данных

Набор данных содержит статические метрики кода, собранные из программных модулей, включая:

*   **Базовые метрики:** количество строк кода (`loc`)
*   **Метрики сложности:** цикломатическая сложность (`cyclo_complexity`)
*   **Метрики Холстеда:** объем (`volume`), сложность (`difficulty`), усилия (`effort`)
*   **Структурные метрики:** количество ветвлений (`branch_count`), условий (`condition_count`)
*   **Целевая переменная:** наличие или отсутствие дефекта (`defect_label`)

## ETL-пакет для обработки и анализа данных о дефектах в программном обеспечении

Проект реализует полноценный **ETL-конвейер**, состоящий из трёх модулей:

- **`extract.py`** — загрузка исходных данных CSV-файла с Google Drive по ID файла
- **`transform.py`** — очистка, нормализация и валидация данных, сохранение в `.csv`
- **`load.py`** — сохранение финальных обработанных данных в CSV формате

### CLI интерфейс

Проект оснащён мощным CLI интерфейсом для гибкого управления ETL процессом:

#### Полный запуск конвейера

```bash
# Использовать настройки из .env (автоматически загружает с Google Drive)
python etl/main.py

# Загрузка с Google Drive напрямую (указав ID)
python etl/main.py --gdrive-id 1kGJHKBwLodGYlnMO_b-_cMvGPDuYC_Fo
```

#### Пошаговое выполнение

```bash
# Только извлечение данных
python etl/main.py --only-extract

# Только трансформация (требует предварительного extract)
python etl/main.py --only-transform

# Только загрузка в финальное хранилище
python etl/main.py --only-load
```


### Примеры использования

#### Пример 1: Первый запуск с Google Drive

```bash
# 1. Создайте .env файл с GDRIVE_FILE_ID
# 2. Запустите полный конвейер
python etl/main.py

```

#### Пример 2: Отладка конкретного этапа

```bash
# Сначала извлекаем данные
python etl/main.py --only-extract --local data/my_data.csv

# Затем отлаживаем трансформацию
python etl/main.py --only-transform

# Наконец загружаем в финальное хранилище
python etl/main.py --only-load
```

## Функциональность ETL

### Extract (Извлечение)
- Загрузка CSV-файла с данными о метриках ПО.
- Поддержка загрузки с Google Drive (через библиотеку gdown).
- Сохранение данных в `data/raw/raw_defect_data.csv`

### Transform (Трансформация)
- Очистка названий колонок: приведение к нижнему регистру, замена пробелов на подчеркивания.
- Удаление дубликатов: поиск и удаление полных дубликатов строк.
- Обработка пропущенных значений:
 - Удаление строк с пропущенной целевой переменной (defect_label).
 - Заполнение пропусков в числовых метриках медианным значением.
- Валидация данных:
  - Удаление строк с отрицательными значениями метрик (например, loc < 0)
  - Фильтрация строк, где defect_label не равна 0 или 1.
- Сохранение результатов в `data/processed/clean_defect_data.csv`

### Load (Сохранение)
- Чтение данных из `data/processed/clean_defect_data.csv`
- Сохранение финальных обработанных данных в `data/final/final_defect_data.csv`
- Вывод итоговой статистики по датасету: общее количество записей, количество и процент дефектных/чистых модулей.

## Анализ данных о дефектах ПО (EDA)

В папке `notebooks/` находится Jupyter Notebook `EDA.ipynb` с полным исследовательским анализом данных, включающим:

### Основные виды анализа

- **Общую информацию о наборе данных** — размер, типы данных, первичный осмотр
- **Статистические характеристики** — среднее, медиана, стандартное отклонение
- **Анализ пропущенных значений** — поиск и обработка пропусков
- **Анализ целевой переменной** — распределение классов (дефектные / без дефектов), выявление дисбаланса
- **Визуализацию распределений** — гистограммы, boxplot, violin plot
- **Корреляционный анализ** — тепловые карты взаимосвязей признаков

### Запуск анализа

```bash
# Запуск Jupyter Lab
jupyter lab notebooks/EDA.ipynb

# Или Jupyter Notebook
jupyter notebook notebooks/EDA.ipynb
```
